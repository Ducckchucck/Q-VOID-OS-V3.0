## 🔥 The "Insane" Upgrade Features

### 1. Self-Healing Polymorphic Engine
The core is no longer static. When a vulnerability or novel attack vector is detected, the **Polymorphic Shell** hooks into an LLM, actively rewriting and patching its own Python source code on the fly. 

### 2. Generative Retaliation & The Solipsism Trap
Instead of just logging attacks, the **MCP Router** hands control to an autonomous LLM agent. Attackers are seamlessly routed into the **Solipsism Trap** :An infinite deception sandbox where the OS dynamically hallucinates fake file systems, databases, and network topologies in real-time to waste attacker resources and study their behavior.

### 3. Hive Mind Swarm Compute
A decentralized P2P Kademlia network that links multiple Q-VOID instances. Beyond sharing threat intelligence, the swarm distributes actual processing workloads (like Grover's search or heavy encryption tasks) across compromised or honeypot nodes, complete with peer **Trust Scoring**.

### 4. Dynamic LLM Cryptography
Beyond standard AES and RSA, Q-VOID features an "insane" crypto layer where an LLM dynamically invents a novel, mathematically unique symmetric encryption algorithm for every payload, returning the custom decryption routine alongside the ciphertext. A true moving-target defense at the algorithm level.

### 5. GhostFS & DNA Steganography
A plausibly deniable encrypted hidden file layer (`GhostFS`) backed by true authentication markers, combined with a DNA Encoder module that translates classified intelligence into biological sequences (A, T, C, G) for extreme stealth.

## ⚡ Quick Start

### 1. Install Dependencies
```bash
python -m pip install -r requirements.txt
```

### 2. Configure LLM Access
Q-VOID OS requires an LLM to power its agentic core. Create a `.env` file in the root directory and add your OpenRouter (or OpenAI) API key:
```env
OPENROUTER_API_KEY=your_key_here
```

### 3. Boot the OS
Launch the interactive terminal shell:
```bash
python -m cli.qvoid
```
Or run the comprehensive test suite to verify all modules:
```bash
python run_all_tests.py
```

## ⚠️ Security & Risks

> **⚠️ WARNING: SELF-MUTATING CODEBASE (EXPERIMENTAL)**
> Phase 3 (Polymorphic Engine) is capable of using Large Language Models to rewrite its own Python source files on the fly to patch vulnerabilities dynamically. This is inherently high-risk. A hallucinated patch or bad mutation could break the system entirely. We implemented this because it's an incredibly powerful capability for an open-source project, but **do not deploy this in a real production environment where uptime is critical.**

- **LLM Code Execution:** Features like the Dynamic Cipher actively execute Python code generated by the LLM (`exec()`). This fits the "moving-target defense" paradigm but introduces severe security risks if the LLM provider is compromised or prompted maliciously.
- **QCrypt Keystores:** Private keys are encrypted at rest. Set `QVOID_KEYSTORE_PASSWORD` in real deployments.
- **GhostFS:** Validates passphrases using an authenticated volume marker. Lose the passphrase, lose the data.
